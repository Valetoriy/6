Турникеты на станциях, терминалы касс -> (выпуск новых транспортных карт,
    продажа поездок, пополнение баланса, продление срока действия ТК)

Конец рабочего для (2:00) -> опрос сервером подключённых к нему турникетов
Сервер получает сведения о входах и выходах со станции за сутки
    -> выявляются карты для занесения в стоп-лист
        | в течении суток сделаны 2 входа подряд, между которыми не зафиксирован выход
        | зафиксирован вход, за которым не зафиксирован ни вход ни выход
        | при входе/выходе используются разные карты

До начала рабочего дня (до 4:59) -> рассылка сервером всем турникетам обновлённой цены поездки
    | администратор сервера даёт об этом команду (и сообщает новую цену)

Администратор обновляет время и дату на сервере -> сервер рассылает турникетам команду синхронизации

Запуск в оборот новой транспортной карты -> сведения об этом приходят от терминала кассы
    -> сервер принимает сведения и сохраняет их в своей БД
    Карта: уникальный номер, тип, срок годности (по истечении не может быть использована для прохода)
    Типы транспортных карт:
        С фиксированным кол-вом поездок
        С неограниченным кол-вом поездок
        Карта-"кошелёк" -> известен её неотрицательный баланс

Пассажир может пополнить количество поездок на карте и/или продлить
срок её годности и/или пополнить баланс карты-"кошелька"
    | карта не находится в стоп-листе
    | тип пополнения соответствуюет типу карты (пример: на карту-"кошелёк" принимаются только пополнения кошелька)
    -> терминал кассы присылает на сервер городского транспорта сведения об операции по карте
    -> сервер сохраняет эти данные для фискального отчёта
    -> сервер производит обновления сведений о карте у себя в БД

На кассе произведена разблокировка карты -> карта удаляется из стоп-листа
Запрос с кассового терминала городского транспорта -> карта окончательно выведена из обращения

Администратор сервера:
    Поддерживает на сервере актуальное состояние о транспортной сети
    Ведёт работу со сведениями о станциях МЦД
    Поддерживает сведения о сети турникетов и кассовых терминалов (добавляя/удаляя/обновляя их)
        Каждый узел этой сети имеет:
            Уникальный IP-адрес
            Тип (турникет/терминал)
            Привязка к одной из станций МЦД

Система должно обрабатывать следующие данные:
    Сведения о станциях МЦД (1-7)
    Сведения об установленных на станциях турникетах и кассовых терминалах (8-11)
    История цены поездки, актуальной в разные периоды времени (12-15)
    Сведения о входах и выходах через турникеты и картах, по которым эти проходы совершены (16-26)
    Текущий стоп-лист, в который занесены транспортные карты (27-30)
    Сведения об операциях, осуществлённых в кассах с транспортными картами (31-48)
