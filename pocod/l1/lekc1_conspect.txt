Формирование требований к АС -> Разработка концепции АС ->
    Техническое задание -> Эскизный проект -> Технический проект ->
    Рабочая документация -> Ввод в действие -> Сопровождение АС

Процессы реализации программных средств:
Процесс анализа требований ПС -> Процесс проектирования архитектуры ПС ->
    Процесс детального проектирования ПС -> Процесс квалификационного тестирования ПС ->
    Процесс комплексирования ПС

ЖЦ ПП:
Определение требований -> Проектирование -> Разработка -> Тестирование -> Документация ->
    Поставка -> Сопровождение

Группы методологий разработки:
    Каскадная (водопадная, waterfall) модель
        Формирование требований -> Проектирование -> Реализация -> Тестирование ->
        Внедрение -> Эксплуатация и сопровождение
    Итеративная (итерационная) модель
        Планирование -> Реализация -> Проверка -> Оценка
    Спиральная модель
        Планирование -> Анализ рисков -> Конструирование -> Оценка результата и при
        удовлетворительном качестве переход к новому витку

Гибкие методологии разработки ПО (Agile-методологии):
    Итеративная разработка
    Динамическое формирование и постоянное изменение требований
    Взаимодействие внутри самоорганизующихся рабочих групп, состоящих из специалистов
        различного профиля

Agile-методологии:
    Scrum
        Жёстко фиксированные и небольшие по времени итерации, называемые спринтами
        Рекулярное предоставление конечному пользователю работающего ПО с новыми возможностями
        Самоорганизация команды, состоящей из небольшого количества специалистов
    XP - экстремальное программирование
        Очень короткие циклы
        Коллективное владение кодом -> парное программирование
        "Заказчик всегда рядом"
        Непрерывная интеграция - рабочие ветки сливаются в общую несколько раз в день,
            постоянные сборки версий
        Разработка через тестирование (TDD)
    Семейство методологии Crystal
        Человеко-ориентированные, а не процессо-ориентированные методологии
        Индивидуальный подбор методологии под проект и команду
        Максимально простые методологии
    Kanban
        Визуализация разработки
        Разделение работы на задачи
        Использование отметок о положении задачи в разработке
        Ограничение работ, выполняющихся одновременно, на каждом этапе разработки
    Lean
        Уменьшение потерь (потери - то, что не добавляет ценности продукта для пользователя)
        Непрерывное обучение и совершенствование
        Позднее принятие решений
        Максимально быстрая доставка пользователю
        Определение потока ценностей
        Мотивация
